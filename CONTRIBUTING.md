# Contributing to CommCare-Cloud

CommCare-Cloud is chiefly developed by Dimagi, but we welcome contributions,
whether it be new tools, simplifications, or documentation improvements.

Before making any commits, make sure you install the git hooks:

```
./git-hooks/install.sh
```

This will make sure you never commit an unencrypted vault.yml file.


## Running tests

To run tests, first install the test dependencies

```
uv sync --group test
```

and then run

```
uv run nosetests
```

Tests include tests of your own specific environments dir.


## Updating dependencies

Dependencies are added to `pyproject.toml`. Dependencies should be unpinned, or
if there is a known range of compatibility, it should be constrained by an upper
and/or lower limit as appropriate.

`uv.lock` is autogenerated using `uv lock` and that contains the exact pinned
versions that are installed with `uv sync`.

Use `uv` to update `uv.lock` to the latest available versions of python
libraries:

```bash
uv lock --upgrade
```

To upgrade a specific package (more common than all at once), use the following
syntax, substituting package name and version:

```bash
uv lock --upgrade-package 'some-package>=1.2.3'
```
